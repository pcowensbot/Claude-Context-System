## ğŸ’¡ Best Practices

### For Developers

**At Session Start:**
- Let Claude read `CLAUDE.md` automatically
- Briefly mention what you want to work on
- Claude will read context and be ready

**During Session:**
- Ask Claude to document important decisions
- Have Claude update CONTEXT.md for major changes
- Trust Claude to follow established patterns

**At Session End:**
- Let Claude create a session log
- Review and commit changes
- Claude will note priorities for next time

### For Claude

(These are built into CLAUDE.md)
- Always read `.claude/CONTEXT.md` first
- Check recent session logs
- Follow documented code patterns
- Ask questions when unclear
- Offer to create session logs
- Update CONTEXT.md when things change

## ğŸ”§ Troubleshooting

### Claude Doesn't Read CLAUDE.md

Make sure:
1. File is named exactly `CLAUDE.md` (capital letters)
2. File is in your project root
3. You mention it: "Read the CLAUDE.md file"

### Context System Not Setting Up

Ask Claude explicitly:
```
"I have CLAUDE.md in my project. Can you read it and set up 
the context system?"
```

### CONTEXT.md Getting Too Large

Claude will suggest solutions:
- Moving detailed docs to separate files
- Archiving old session summaries
- Creating topic-specific context files

### Session Logs Piling Up

Add to your project's maintenance routine:
```bash
# Archive logs older than 30 days
mkdir -p .claude/archive
mv .claude/sessions/$(date -d '30 days ago' +%Y-%m-*)*.md .claude/archive/
```
